I want to enhance the “Existing Experts” invitation flow in the Projects module of the Mirae Connext app.

Goals:
1) For each attached expert in a project, I want to:
   - Generate a project-specific invite link (as we already specified).
   - Choose one or more channels: Email and/or WhatsApp.
   - Choose a language: English, Portuguese, or Spanish.
   - Get a ready-to-use invitation message (with the link included) that I can copy and paste.

2) I want both Email and WhatsApp templates to support 3 languages:
   - EN (default)
   - PT-BR
   - ES

3) For now, I don’t need direct integration with email servers or WhatsApp API.
   - Copy-to-clipboard from the UI is enough.

Please implement the following.

--------------------------------------------------
A) DATA & INVITE LINK REUSE
--------------------------------------------------

We already have (or should have) the mechanism to generate a project-specific invite link for an existing expert:

- POST /api/projects/:projectId/experts/:expertId/invite-link
  - Returns an object with the full invite URL, e.g.:
    {
      "inviteUrl": "https://portal.miraeconnext.com/expert/project-invite/<token>"
    }

If a token already exists and is active, reuse it instead of creating a new one.

Keep that behavior and use this inviteUrl in the templates below.

C) UI – MULTI-CHANNEL & MULTI-LANGUAGE INVITE CONTROL
--------------------------------------------------

In the Project Detail → Existing Experts table:

For each expert row, add a button:
- “Invite”

When the PM clicks “Invite”, open a small modal or side panel with:

1) Language selection:
   - Label: “Language”
   - Options:
     - English (en)
     - Português (pt)
     - Español (es)
   - Default:
     - Use the project’s default language if available, otherwise "en".

2) Channel selection (multi-checkbox):
   - Label: “Channels”
   - Checkboxes:
     - [ ] Email
     - [ ] WhatsApp

3) Action buttons:
   - “Generate Message”
   - “Close”

Behavior when “Generate Message” is clicked:

- Call POST /api/projects/:projectId/experts/:expertId/invite-link
  - Get inviteUrl

- Use:
  - projectTitle from project
  - expertName from expert
  - pmName from the logged-in PM user
  - language from the selector
  - inviteLink = inviteUrl

- Based on selected channels:
  - If Email is checked:
    - Build:
      - subject = inviteTemplates.email[lang].subject
      - body    = inviteTemplates.email[lang].body
    - Replace placeholders {{expertName}}, {{pmName}}, {{projectTitle}}, {{inviteLink}}.
  - If WhatsApp is checked:
    - Build:
      - body = inviteTemplates.whatsapp[lang].body
    - Replace the same placeholders.

- In the modal, show a result area with:

  - If Email is selected:
    - A section titled “Email Invite”
    - A read-only text field or textarea with:
      - “Subject: …”
      - A blank line
      - “Body: …”
    - A “Copy Email Invite” button that copies the entire email block to the clipboard.

  - If WhatsApp is selected:
    - A section titled “WhatsApp Invite”
    - A read-only textarea with the WhatsApp text.
    - A “Copy WhatsApp Message” button that copies just that message.

- Show a small toast notification after each copy:
  - e.g. “Email invite copied to clipboard.”
  - e.g. “WhatsApp message copied to clipboard.”

--------------------------------------------------
D) PERMISSIONS
--------------------------------------------------

- Only logged-in users with role = "pm" or "admin" should see and use the “Invite” button for existing experts.
- RAs should not have access to this PM invite modal for existing experts.
- RA invitations for new experts are handled separately via RA-specific invite links.
